# Mission System — DSC

## Overview

Missions are generated by combining a **populated AO** (guards, garrison, patrols) with a **mission objective** (HVT, sabotage target, etc.). Currently only Kill/Capture is implemented.

## AO Population (`fnc_populateAO`)

Takes a location object from `fnc_scanLocations` and fills it with enemy forces. Mission-type agnostic — objectives are layered on top.

### Inputs

- `_location` — Location hashmap from scanner (has structures, tags, position)
- `_classifiedGroups` — Pre-classified group data from faction pipeline
- `_config` — Optional overrides (faction, side, density, patrol count)

### Density Auto-Detection

If not overridden, density is derived from location tags:

| Location Tag | Density |
|-------------|---------|
| `high_density` | heavy |
| `medium_density` | medium |
| `low_density` | light |

### Spawn Layers

#### 1. Guards (`fnc_setupGuards`)

Two phases:

**Phase 1 — Dedicated structures** (military only):
- Finds guard towers, bunkers, patrol towers via `isKindOf` checks
- Top position of each structure: static weapon (HMG/GMG/AT/AA) or lookout
- Open sky check (`lineIntersectsSurfaces`) determines if static can be placed
- Static weapon selection biased toward HMG/GMG (70%) over launchers

**Phase 2 — Perimeter security** (military + civilian):
- Generates evenly-spaced positions around structure cluster edge
- Searches for concealment (trees, walls, bushes) within 20m of each point
- Spawns sentry pairs (2 units) — one facing outward, one facing inward
- 3-5 perimeter positions per location

All guard units get `disableAI "PATH"` and `fnc_addCombatActivation`.

#### 2. Garrison (`fnc_setupGarrison`)

Uses **Anchor + Satellite** model:

- **Anchors**: Main structures (multi-story, barracks, HQ buildings) — selected spread apart
- **Satellites**: Nearby side structures within 50m of each anchor

Density profiles control spawning:

| Density | Anchors | Groups/Anchor | Satellites/Anchor | Position Fill |
|---------|---------|---------------|-------------------|---------------|
| Light | 2-4 | 1 | 0-1 | 50% |
| Medium | 3-5 | 1-2 | 1-2 | 70% |
| Heavy | 4-6 | 1-2 | 2-3 | 90% |

Per-building caps: main structures = 4 units, side structures = 2 units.

Groups are spawned from CfgGroups templates via `BIS_fnc_spawnGroup`, then distributed across building positions.

#### 3. Patrols (`fnc_setupPatrols`)

- Count scales with density: light 1, medium 1-2, heavy 2-3
- Spawn radius: proportional to location radius (100-300m out)
- Patrol radius: spawn radius + 50-100m
- 15% chance of special group (AT or AA team) per patrol slot
- Patrols walk waypoints around the location center

### Output

```sqf
_aoResult = createHashMapFromArray [
    ["location",      _location],
    ["groups",        [all spawned groups]],
    ["units",         [all spawned units]],
    ["vehicles",      [static weapons etc.]],
    ["defenderUnits", [guard + garrison units]],
    ["patrolGroups",  [patrol groups for QRF]],
    ["garrisonUnits", [garrison units for HVT placement]],
    ["tags",          [doctrine tags per group]]
];
```

## Combat Activation (`fnc_addCombatActivation`)

All defenders start with `disableAI "PATH"` — they hold position until combat begins.

Trigger: `FiredNear` event handler on every unit in the group.
- 0.5s reaction delay
- Enables PATH AI
- Removes all EHs from group (performance cleanup)

This means players can observe the AO without triggering movement until the first shot.

## Kill/Capture Mission (`fnc_generateKillCaptureMission`)

### HVT Placement

1. **With bodyguards** (preferred): Find a garrison unit whose building has ≥3 positions. Place HVT in a free position in that building, join the garrison group.
2. **Alone in structure** (fallback): Pick any structure with building positions, create new group.
3. **Open ground** (last resort): Spawn at location center.

HVT class: searches faction's CfgVehicles for officer/commander/leader classname.

HVT gets:
- `DSC_isHVT = true` (public variable)
- `DSC_hvtName` — random target designation
- `disableAI "PATH"` + `setUnitPos "UP"`

### Mission Marker

Red `Contact_circle4` marker at HVT building or location center, labeled `"HVT: <locationName>"`.

### Mission Data

```sqf
_mission = createHashMapFromArray [
    ["type", "KILL_CAPTURE"],
    ["location", _locationPos],
    ["locationName", _locationName],
    ["locationTags", _locationTags],
    ["entity", _hvtUnit],
    ["entityBuilding", _hvtBuilding],
    ["groups", _aoGroups],
    ["patrolGroups", _patrolGroups],
    ["defenderUnits", _defenderUnits],
    ["units", _aoUnits],
    ["vehicles", _aoVehicles],
    ["marker", _targetMarker],
    ["startTime", serverTime],
    ["status", "ACTIVE"]
];
```

Stored globally as `DSC_currentMission`.

## Mission Briefing (`fnc_createMissionBriefing`)

Creates an Arma task with intel-style text. Briefing includes:

- **Relative location** — "in Kavala" or "northwest of Agios Dionysios"
- **Area description** — derived from location tags (urban, settlement, isolated military position)
- **Troop estimates** — fuzzy, not exact (fireteam/squad/platoon-sized)
- **Patrol activity** — light/moderate/heavy
- **Threats** — AT teams, AA capability, static emplacements (from AO doctrine tags)
- **ROE** — Weapons free

Task is display-only (no map icon) — the marker handles map visualization.

## Patrol Convergence (`fnc_convergePatrols`)

When combat activates, patrol groups in the area converge toward the mission area as QRF. Uses `patrolGroups` from the AO data.

## Mission Cleanup (`fnc_cleanupMission`)

Deletes:
- All tracked units (including dead bodies)
- All tracked vehicles
- All groups
- Mission marker
- Player drop markers
- Clears `DSC_currentMission` global

Uses `sleep 0.05` between deletions to avoid frame spikes.

## Mission Debrief (Client-Side)

Player interacts with `jointOperationCenter` flagpole → "Debrief Mission" action:
- Checks all mission groups via `fnc_groupActive`
- If no groups active → `missionComplete = true`
- Sets `missionInProgress = false`

## Influence System (`fnc_initInfluence`)

### Military Tiers (from scanner)

The scanner assigns a `militaryTier` to each military anchor based on military structure count:

| Tier | Mil Structures | Role |
|------|---------------|------|
| `base` | 8+ | **Occupation zone** — always faction-controlled, generates influence |
| `outpost` | 4-7 | **Satellite** — controlled if near a base, contested otherwise |
| `camp` | 1-3 | **Contention point** — lightly held or neutral, prime mission targets |

### Location Tiers (influence classification)

| Tier | Source | Role |
|------|--------|------|
| Bases | Military anchors with tier `"base"` | Direct faction assignment via campaign profile |
| Outposts | Military anchors with tier `"outpost"` | Inherit from nearest base within 4km, or contested |
| Camps | Military anchors with tier `"camp"` | Mostly neutral, small chance of light contested presence |
| Populated Areas | `NameCityCapital`, `NameCity`, `NameVillage` | Receive influence from nearby bases + outposts within 3km |
| Mission Sites | Everything else (NameLocal, etc.) | Inherit from nearest influence source |

### Campaign Profiles (base assignment only)

| Profile | OpFor Bases | BluFor Bases | Contested |
|---------|------------|-------------|-----------|
| `offensive` | 80% | 10% | 10% |
| `defensive` | 25% | 65% | 10% |
| `contested` | 50% | 30% | 20% |

### Camp Occupation Chance

| Profile | Chance of light contested presence |
|---------|-----------------------------------|
| `offensive` | 30% (opForPartner or irregulars) |
| `defensive` | 10% |
| `contested` | 20% |

### Influence Propagation

1. Bases assigned directly via campaign profile ratios (high influence 0.7-1.0)
2. Outposts inherit from nearest base within 4km (scaled by distance, 80% strength)
3. Camps mostly neutral, small chance of light contested (irregulars/partners)
4. Player base markers force bluFor (1.0 influence)
5. Populated areas: weighted average of nearby bases + outposts within 3km
6. Mission sites: inherit from nearest base, outpost, or populated area

### Output

```sqf
_influenceData = createHashMapFromArray [
    ["bases", [...]],
    ["outposts", [...]],
    ["camps", [...]],
    ["populatedAreas", [...]],
    ["missionSites", [...]],
    ["influenceMap", {
        "loc_0": { controlledBy: "opFor", influence: 0.85, type: "base", faction: "OPF_F" },
        "loc_5": { controlledBy: "opFor", influence: 0.5, type: "outpost", faction: "OPF_F" },
        "loc_12": { controlledBy: "neutral", influence: 0, type: "camp", faction: "" },
        "loc_20": { controlledBy: "bluFor", influence: 0.7, type: "populatedArea", faction: "" },
        ...
    }],
    ["locations", [enriched hashmaps]],
    ["campaignProfile", "offensive"]
];
```

`fnc_updateInfluence` exists for feeding mission results back (not yet wired into loop).
