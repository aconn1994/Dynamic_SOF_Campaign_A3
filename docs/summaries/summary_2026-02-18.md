Building an Arma 3 mod framework for dynamic special operations missions. Uses CBA XEH for function registration and
HEMTT for building. The goal is to create a "skilled zeus in a box" that combines singleplayer campaign immersion

## Roadmap Discussion

Reviewed  examples/roadmap_draft.md  and discussed Phase 2 (Mission Types). Key design decisions:

Mission Generation Pattern:

1. Pick mission type → determines valid location types
2. Pick location → from valid types
3. Generate mission entity → HVT, hostage, objective object, etc.
4. Populate area → guards/garrison/patrols (density varies by mission)
5. Create briefing → mark objective, describe task

Proposed Mission Data Structure:

  private _mission = createHashMapFromArray [
      ["type", "KILL_CAPTURE"],
      ["location", _selectedPos],
      ["locationType", "compound"],
      ["entity", _hvtUnit],
      ["entityBuilding", _building],
      ["intelObjects", []],
      ["density", "medium"],
      ["briefing", "..."]
  ];

Design Philosophy:

• Soft objectives - tasks guide, not dictate
• Intel as currency - every location has potential intel
• Resource pressure - forces player tradeoffs
• Layered truth - briefing is best-guess, reality may differ
• Arma task system as display layer only, not control flow
